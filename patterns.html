Accessibility

To make our content accessible to everyone, we need to pay special attention to how that content is marked up. This means, in WordPress, using the text editor to add content, and following some simple guidelines (as below).

Textual Content Markup

  • Use semantic and hierarchical HTML and strategies, such as:
    • headings to introduce each content block
    • lists to create listed items
    • paragraphs for paragraphs, not line breaks, divs or text without markup
    • inline elements like strong, em, cite, et cetera, to meaningfully identify text
    • unique, descriptive titles for pages that orient the pages in the site
    • linked text should be descriptive outside of the page context, without verb phrases
  • DO NOT use:
    • inline styles; use defined classes as outlined in Special Cases
    • nested lists more than three levels deep
    • deprecated markup
    • line breaks for formatting paragraphs or lists
    • verb phrases in linked text, including "click here" (see References for more)
    • colour alone to convey meaning

Paragraphs and Tags Used With Paragraphs

Paragraphs are within the <p></p> tags. Within a paragraph, you can markup the content by adding links, citing Documents, emphasizing key words and adding code. You can also use superscripts1 and subscripts2. If you need more advanced markup, please see the References provided.

Paragraph tag: <p></p>
Hyperlink tag: <a title="If the link doesn't contain the Title of the Linked-to Page, you must put it here to make the link accessible." href="http://somelink.com/">Title of Linked-to Page</a>
Cite tag: <cite></cite>
Strong emphasis tag: <strong></strong>
Stress emphasis tag: <em></em>
Code tag: <code></code>
Superscript tag: <sup></sup>
Subscript tag: <sub></sub>
Line break tag: <br>
Prevent word break tag: <wbr> (Sample: Goes<wbr>between<wbr>words when you don't want a line-break to separate them. Use sparingly!)

Ordered Lists

For use when the items need to be ordered. Our default is list-style-type: decimal;, but we can create other versions if needed.

  1. decimal (1, 2, 3...)
  2. decimal-leading-zero (01, 02, 03...)
  3. lower-alpha (a, b, c...)
  4. lower-roman (i, ii, iii...)
  5. upper-alpha (A, B, C...)
  6. upper-roman (I, II, III...)

	<li>list item</li>

</ol>

Definition Lists

Term
Definition or description of the term.
Definition Lists
A definition list is a list of terms and corresponding definitions. Definition lists are typically formatted with the term on the left with the definition following on the right or on the next line. The definition text is typically indented with respect to the term.
Usage
Since the term and the definition/description are closely linked, this helps alternative readers understand the relationship between the term and its definition/description.
<dl>
	<dt>Term</dt>
	<dd>Definition or description of the term.</dd>
	<dt>Definition Lists</dt>
	<dd>A definition list is a list of terms [...]</dd>
	<dt>Usage</dt>
	<dd>Since the term and the definition [...]</dd>
</dl>

Using Width Classes

There are a number of custom widths already set for your use. You can use them with any block-level element, such as div, p, aside, table, ul, ol, dl.

The only thing that these classes effect is the width. The rest of the styling is applied to the element or other classes listed.

class="half"
class="threequarters"
class="full"

Using Multiple Classes

You can list more than one class name in the class attribute of any element. This allows you to add multiple conditions to your element.

<aside class="note quarter center">
<p>This is an aside with the classes note, quarter, center applied.</p>
</aside>

Deprecated Tags (please do not use)

Instead of a horizontal line, which is created with this tag: <hr>, please use the logical heading element for the content ( <h1>, <h2>, <h3>, <h4>, <h5>, <h6>) instead, to convey semantic meaning.
